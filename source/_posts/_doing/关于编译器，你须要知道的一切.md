# 1 GCC/G++

## 1.0 简介

GCC和G++都是一个叫GNU的组织搞出来的，现在是最常用的C/C++编译器，木有之一。其中，G++只能用来编译C++语言，而GCC还可以编译C语言。同时，它们也是支持C++语言特性最丰富的编译器（[可以参考这里](https://zh.cppreference.com/w/cpp/compiler_support)）。

## 1.1 C++代码/指令
`#pragrma GCC Optimize(x)`：x是几就开几级优化。x可以为0（默认）、1、2、3，数字越大优化的越多，编译速度越慢，**一般情况下（不保证）**运行的速度越快。

?> Tips：一般STL里的东西都比较好氧，即加了这行代码之后运行速度比不加有极大的提升。

!> Notes：正因为它优化效果不低，部分算法竞赛和评测工具不允许选手手动开启O2优化，请遵循参加的赛事规则/评测工具（网站）的规定。

## 1.2 GCC/G++编译指令

这俩编译指令一样，~~为了偷懒~~下文统称GCC。鉴于实在是太多了，只讲大部分比较常用的

?> Tips：下文指令请在终端中运行。

`gcc -v`：查看GCC版本。

`gcc aaa -o bbb -std=ccc -Dddd -Oe`：以ccc作为语言标准编译aaa这个源文件（一般C语言的源文件扩展名为“.c”，C++的在Windows下为“.cpp”，Linux下为“.cc”，但cpp和cc其实一样）并开启Oe优化，生成的可执行文件名为bbb，编译时定义ddd宏。由于几乎编译每份代码都要用这个指令，下面把它起个名字（**仅在本文内有效**，离了本文就忘掉这名字吧~）叫：“基准指令”。

其中除被编译文件外（废话，不然GCC哪儿知道你要编译啥？）都可以省略，默认bbb为'a.out'（Linux系统下），ccc默认版本跟GCC版本有关（想自行指定见附录·表A），e默认是0（O0优化就是没有优化），ddd可以自行指定（一般是定义`ONLINE_JUDGE`宏，可以结合`#ifndef`来判断是否在OJ中），没有就真的没有了……

<details>
<summary>讲点别的·GCC是咋编译的啊？</summary>
GCC这玩意儿是咋编译的？

这就是大名鼎鼎的“编译原理”，具体过程如下：

首先先进行一个预处理（也叫预编译）操作，就是将那些以“#”开头的预处理指令处理了、将注释删掉（其实不会改源文件，注释是被无视了）。

比如`#include` 会将被include的文件简单粗暴的替换到文件的开头（如果它还include了别的文件，那么先将别的文件加进来，再加入它自己。就这样递归处理到某个文件啥也不include了才停下。所以使用万能头写起来快编译非常慢，就是因为此刻可怜的编译器在玩儿命递归）而你那点儿~~少的可怜~~的代码就会~~孤苦伶仃、弱小可怜又无助~~的丢在最后面。

`#define`（宏定义）操作也会则会简单粗暴的把你代码中用到的宏替换成define后面的那坨，如果有参数就先把参数代入了再替换。

`#ifdef xxx`=if(定义了xxx这个宏) 执行后面的代码

`#ifndef xxx`相当于if(没定义xxx这个宏) 执行代码

`#if`、`#else`、`#elif`、`#endif`：逻辑判断，给示例代码，看看就懂了~~吧~~：
```cpp
#if 3 > 2 // 相当于if(3 > 2)
	cout << 1;
#elif 3 == 2  // 相当于else if (3 == 2)
	cout << 2;
#else  // 相当于else
	cout << 3;
#endif  // 预处理指令中的判断都要在最后写endif
#ifdef ONLINE_JUDGE // 判断是否定义了“ONLINE_JUDGE”这个宏
	cout << 4;
#endif
#ifndef ONLINE_JUDGE // 判断是否没定义“ONLINE_JUDGE”这个宏
	cout << 5;
#endif
```

其它预编译指令（也不太咋用了）的见[这里](https://blog.csdn.net/wulei202/article/details/7636654)。

预编译指令在“预编译”这步完成后自己也没了。~~好伟大啊。~~

可以在基准指令的aaa前面加上“-E”来记录这个过程，一般输出文件扩展名为“.i”。

然后编译。编译会进行一定的优化，并将你的代码转换为汇编代码。

{% spoiler 什么是汇编？}
汇编代码的执行速度很快（虽然比不上二进制指令（专业术语：“机器码”），但已经比C/C++、Java、Python这种“高级语言”快多了。然鹅，汇编语言（也称为“符号语言”）和机器码一样，几乎每台机器都不一样，而且非常复（qi）杂（pa），高级语言就是为了可移植性（即一个机器上的代码转移到另一台机器上仍然可以正常运行）和可读性（别人读你的代码能读懂，不会吐血，其实就是相对简单）而发明的）
{% endspoiler %}

[具体优化见OI Wiki](https://oi-wiki.org/lang/optimizations/#%E7%BC%96%E8%AF%91%E5%99%A8%E4%BC%98%E5%8C%96%E7%AE%80%E4%BB%8B)~主要就是把乘除常数改成位运算、消除执行了跟没执行一样的代码（比如`while(0)`里面的玩意儿）之类的，不会改变代码意思，但是会优化运行速度。O2、O3之类的就是优化的多了一些。

这个部分比较熟悉了，你的语法错误就是在这儿被发现的。C++语言标准就是在指定编译器遇到什么样的代码该编译成啥样子。至于未定义行为（比如`i = i ++ + ++i`这种奇葩）由各大厂商自行处理。

{% note warning} 
Warning：鉴于未定义行为在不同的编译器甚至是同一个编译器的不同模式中都有可能会产生不同的结果，请**切记不要**写出这种代码。
{% endnote %}

可以在基准指令的aaa前面加上“-S”来体现这个过程，一般输出文件扩展名为“.s”。

第三步是汇编，即将汇编代码转换为机器码。

最后链接。汇编完的机器码直接运行会完的（……究其原因是有时一堆文件会合在一起作为一个项目编译，这时有些源文件会使用别的源文件的内容，然后，然后就炸了……

链接方式分为动态链接（默认）和静态链接两种。动态链接是默认情况，无需设置。静态链接需要在基准指令后面加上`-static`后缀。动态链接有时能节约内存，且最后生成的可执行文件会小点，但有时会造成性能损失，而静态链接则相反。

`-Wall`：显示所有警告（默认会有，但只有一部分，也可以用`-Wextra`显示多点，但`-Wextra`数量没`-Wall`多，因此直接用`-Wall`得了）。警告里的内容一般不会让你产生编译错误（除非你加了`-Werror`把警告干成了错误），但很可能会产生逻辑错误。

`-pedantic`：帮你发现一些不符合语言标准的代码，但不全。这个会结合你的语言标准设定来。

`-lm`：链接数学（cmath）库。不链写了`#include<cmath>`还是会炸……
